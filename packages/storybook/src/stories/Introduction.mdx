# Tagaddod Design System

Welcome to the Tagaddod Design System documentation!

## Overview

This design system provides a comprehensive set of React components built with:

- **Design Tokens**: W3C-compliant tokens for consistent theming
- **Tailwind CSS v4**: Automated utility generation from design tokens
- **TypeScript**: Full type safety
- **Multi-brand Support**: Tagaddod and GreenPan themes

## Token-based Utilities

Our system uses two types of Tailwind utilities:

### Standard Utilities
These map to common Tailwind class names but use our design tokens:

```jsx
<div className="p-4 bg-purple-1200 text-white rounded-lg">
  Content
</div>
```

### Token-specific Utilities
These use the `t-` prefix for direct token access:

```jsx
<div className="p-t-400 bg-t-surface-background text-t-primary rounded-t-200">
  Content
</div>
```

## Component Architecture

All components are built with:

1. **Token-based styling**: All visual properties reference design tokens
2. **Accessibility**: WCAG 2.1 AA compliance
3. **Responsive design**: Mobile-first approach
4. **Multi-theme support**: Automatic theme switching

## Getting Started

### Using Components

```jsx
import { Button } from '@tagaddod/react';
import '@tagaddod/react/dist/styles.css';

function App() {
  return (
    <Button variant="primary" size="md">
      Click me
    </Button>
  );
}
```

### Using Tokens

You can reference any design token through utility classes:

```jsx
// Spacing
<div className="p-t-400 m-t-200">Content</div>

// Colors
<div className="bg-t-surface-background text-t-text-primary">Content</div>

// Typography
<div className="text-t-heading-md font-t-semibold">Heading</div>

// Borders
<div className="border-t-200 rounded-t-300">Content</div>
```

## Design Tokens

Our design tokens are organized into:

- **Primitives**: Base values (colors, spacing, etc.)
- **Semantics**: Contextual values (action colors, surface colors, etc.)
- **Components**: Component-specific tokens

All tokens follow the W3C Design Token format and are available as CSS custom properties with the `--t-` prefix.
